Fable 如何剪辑自己的发布视频 — Thariq (Cloud Code)

大家好，我是来自 Cloud Code 的 Thariq。我收到了很多关于 Fable 是怎么剪辑自己发布视频的问题，所以我来讲一下。你们知道吗，这个演示文稿也是它做的，所以有点套娃。它也在剪辑这个视频。没错，它确实剪辑了这个发布视频，那它是怎么做到的呢？

我和一个视频制作团队合作，我们拍了 17 条素材。一共有四个场景，每个场景拍了好几条。这就是我拿到的原始素材——没有调色，25GB 的文件，最后变成了这个成品。调完色之后我们加了一些 UI，把它们拼接在一起，所有这些都是 Cloud Code 完成的。我没碰过终端，也没碰过任何视频编辑器。

那它具体是什么样的呢？起始提示词是这样的：

"我有一堆录制的视频，我想让你对它们运行一个描述服务，然后把最好的镜头拼接成一个最终剪辑。注意，每个场景有多条素材。通常最好的素材是后面几条，因为嗯啊比较少，但也不绝对。我还重拍了最后一个场景。"

然后我让它创建一个 JSON 文件来描述视频——每个场景用哪个片段。我只给了它一个视频文件夹，我没有告诉它哪些视频属于哪个场景。

它做了什么呢？它用 Whisper 转录了每一条素材，给每个视频创建了一个单词数组，然后去重并选出最好的。所以从 17 条素材中，它自动识别出哪些属于哪个场景，然后找到最合适的那条。

它把所有这些编译成一个编辑 JSON 文件——包括每个场景、候选素材、选择原因，以及每个场景的开始和结束时间。然后用 FFmpeg 把第一版剪辑拼接起来。

然后是调色。原始画面是中性 709 色彩空间的，看起来很平淡。我对调色一无所知，我只知道调色是个事儿。所以我就让它写了一堆调色方案，它就把原始画面调成了最终的色彩风格。

下一步是做设计素材。我之前给了它脚本，需要做一堆设计资产。然后我说用 Remotion——Remotion 是一个库，它可以把视频和 React 组件结合在一起，导出成最终视频。

所以我让它做了一堆 Remotion 组件来实现设计。全是 React 代码。它根据之前的转录 JSON 来选择节拍和时间点，控制淡入淡出，有各种可调参数。最终剪辑就是——什么时候显示哪个 UI 组件。比如我说到"Cloud 做的对不对"的时候，它就在这个节拍点切换了两个组件。这些全是它自己根据转录和 Remotion 组件自动完成的。

然后我说把设计导出到 Figma，让设计团队能编辑。第一版是 Cloud 风格的初稿，但设计团队的品味比我好得多。我还给了他们一个 HTML 文件让他们调整调色参数。他们编辑完 Figma 后还给我，我用 Figma MCP 直接说："设计在这个 Figma 文件里更新了，你能把视频更新成匹配的样子吗？" 然后它就做到了——变成了设计团队做的那种干净透明的风格。

重点是：这一切都只是代码。Cloud Code 写的所有代码包括：
- 每个视频的转录 JSON
- 最终编辑 JSON 文件
- 调色文件
- 一堆 Remotion 的 React 组件
- 最终拼接编辑

然后用 npx remotion render 渲染视频、应用调色、叠加 Remotion UI，最终合成。

就是这样做到的。Fable 真的很厉害。现在剪辑变简单了，我希望能做更多视频。有问题的话告诉我，谢谢大家。
