# 2026-07-18 功能迭代记录

本次迭代对应发布版本 `v0.6.0`，开发分支为 `codex/node-output-cache`。重点是将可视化工作流升级为通用的端口化 node runtime，并完善 AI 生成、自定义工具、节点缓存恢复、科技口播生产链路和长任务稳定性。

## 1. Node runtime 与数据传递

### ComfyUI 风格端口

- 节点使用 `NodeInput` / `NodeOutput` 声明输入输出端口。
- 工作流边保存 `source_handle` / `target_handle`，不再只依赖节点级 `source` / `target`。
- 支持 `*` 通配类型和 `multi=true` 多上游输入。
- 前后端校验端口是否存在、类型是否兼容，以及非 multi 输入是否重复连接。
- 历史 `reads` / `writes` 和 `ctx.xxx` 保留兼容，但不再作为新节点的主要契约。
- 历史无 handle 工作流加载时会推断端口，并在迁移前生成备份。

### 节点输出缓存

- 新增 `server/engine/node_cache.py`，缓存目录为：

  ```text
  data/<date>/.node_cache/<workflow_id>/<node_id>/<cache_key>/
    manifest.json
    output.pkl
    files/
  ```

- 缓存 key 包含节点类型、节点配置、输入数据和工作流边绑定。
- 节点成功后缓存结构化输出、旧兼容字段和声明的本地产物目录。
- 再次运行命中缓存时恢复端口输出和文件，并发送 `node_cached` 事件。
- 支持从指定节点恢复运行：上游节点只恢复缓存，目标节点及下游继续执行。
- 属性面板新增“从此节点恢复运行”和“清除节点缓存”。
- 清除新节点缓存后不会误命中旧式“输出目录存在”缓存，保证节点真正重跑。
- `force_no_cache` 会跳过读取缓存，但成功执行后仍刷新缓存。

## 2. AI 生成工具、节点和工作流

### 标准生成 Agent

- 工具、节点和工作流的 AI 创建统一使用 generation agent 循环。
- Agent 会执行生成、结构解析、规则校验、错误反馈和自动修复，不再只进行单轮文本生成。
- 生成用模型由用户从全局 `text` / `coding` 模型池选择，与节点运行时模型分离。

### 自定义工具

- 自定义工具切换为 `SKILL.md + Python 执行器` 模式。
- 表单支持直接编辑 Skill 文档和 Python 代码。
- AI 创建工具可选择模型，并经过结构与代码校验。
- 新增工具注册和执行 API，以及基础 Python runner 支持。
- 弹窗点击遮罩不再直接关闭，避免未保存内容丢失。

### 自定义节点

- AI 生成节点可选择参考类型：普通处理、模型调用、Agent、Trigger 或 Listener。
- Prompt 注入项目概况、节点目录、已注册节点、参考节点摘要、关键代码片段和结构约束。
- 未选择参考节点时，由 LLM 根据任务确定节点类型。
- Agent 节点可从自定义工具中选择工具白名单；无工具时可生成纯 LLM Agent。
- 模型调用和 Agent 节点的属性面板使用全局模型下拉框。
- 新生成的 Processor/Agent 必须声明端口、通过 `self.get_input()` 读取输入并返回 dict 输出。
- Agent 生成规则包含停止标记检查、最大轮数、JSON 修复、工具失败处理和最终输出校验。
- 支持代码编辑和 AI 编辑已有自定义节点。

### 自定义工作流

- AI 工作流先生成自然语言计划和端口级节点图草案，用户确认后才保存。
- 预览展示节点端口、连线 handle、类型冲突和 multi 约束问题。
- 校验失败的结果可保存为草稿，并在列表和编辑器中展示具体原因。
- 草稿存在校验问题时禁止直接运行。

## 3. 工作流执行稳定性

- 停止操作会同步停止标记、取消运行任务并终止节点创建的子进程。
- 运行历史中的遗留 `running` 状态会与当前真实运行状态对账。
- 节点失败后取消同层未完成任务，避免工作流继续错误推进。
- 修复同步 `restore_cache()` 被错误 `await` 的问题。
- 所有长任务通过 `on_progress` 及时上报进度；执行器每 5 秒发送心跳。
- `on_progress` 支持工作线程安全回调。
- 修复 agents 日志重复挂载导致同一日志打印多次的问题。
- 修复 `run_server.py` 和执行器中的中文乱码注释及运行提示。

## 4. 科技口播内容链路

### 采集与素材

- Collect 节点可通过属性选择 AI 科技或今日热搜模式。
- GitHub Trending 尽量完整抓取，不再固定只保留少量项目。
- GitHub README 会下载并作为科技口播的事实输入；无有效 README 的条目跳过，不阻断工作流。
- 图片识别按文件后缀过滤不支持的格式，避免 Vision API 因格式错误中断。

### 科技口播 Agent

- 新增 `tech_broadcast_script_agent` 社区节点。
- 综合采集数据、下载素材、图片识别、音频转录和 README 内容生成口播脚本。
- 默认选择 10-20 个有效主题，素材不足时覆盖全部有效主题。
- 每个主题要求包含概览、机制、证据、使用场景、影响、限制和自然过渡。
- 每句话以中文句号为边界形成一个 voice 元素，不再按字符数硬拆句。
- 每个主题最后一句必须承担“下一条”过渡作用。
- 口播文本不由 LLM 生成方言控制指令，VoxCPM 控制指令由节点配置统一添加。

### VoxCPM2 TTS

- 增加 WSL VoxCPM2 服务自动启动和健康检查。
- 新增极致克隆服务脚本，缓存参考音频 prompt context。
- 默认参考文本和参考音频统一进入项目配置。
- TTS 属性新增“是否定制”；关闭时 `control_instruction` 为空，开启时使用可配置方言/语速指令。
- TTS 输出同时包含音频数据和回填真实时长、起始时间、文件路径后的脚本。
- 保留 Fish Audio 节点与执行器作为可选实现，默认科技口播工作流使用 VoxCPM2。

## 5. 配音后编排、字幕与画面

- 新增 `voice_visual_director_agent`，根据最终口播脚本和真实音频时长生成其他轨道。
- voice 轨和口播正文保持不变，只编排 visual、overlay、remotion、live2d、background 等轨道。
- 素材展示前后必须回到演播室，素材片段限制时长和占比，并使用淡入淡出转场。
- 素材展示期间 Live2D 缩小到角落，其他时间恢复演播室主播画面。
- 新增独立 Subtitle 节点，生成 ASS 字幕轨并在最终合成的最上层渲染。
- Visual 图片保持完整宽高比，使用背景填充，不再简单放大裁掉主体。
- Overlay 与 Visual/Live2D 分离渲染，避免轨道内容互相覆盖。

## 6. 最终合成优化与故障处理

- 2026-07-16 的长视频合成确认发生 FFmpeg `0xC0000005` 原生内存访问冲突。
- 原实现对 1080x1920 全时长视频执行逐像素 `blend`，同时产生约 397 个线程和较高内存占用。
- 新实现只截取实际素材时间窗口，使用 `trim + alpha fade + overlay` 完成片段级转场。
- 限制 VP9 解码、复杂滤镜和 x264 编码线程，降低线程爆炸与 FFmpeg 崩溃概率。
- 使用 `-progress` 实时上报百分比、帧数、fps 和速度。
- FFmpeg 失败时记录十六进制退出码、最后进度和独立错误日志。
- 输出先写入 `.part.mp4`，通过 `ffprobe` 校验时长和容器后才替换正式文件。
- 失败残片不会计入 `FinalData`，节点会正确失败并使工作流停止。
- 合成缓存版本升级，旧损坏缓存不会再次命中。

## 10. 发布与使用说明

- 后端以 `reload=False` 运行，更新后必须重启 `python run_server.py`。
- 2026-07-16 失败生成的 `tech_broadcast_script.mp4` 是不可播放残片；重启后从“最终合成”节点恢复运行即可，无需重新生成上游音频和渲染轨。
- 本次提交不包含 `settings/global.json` 的本地密钥修改，也不包含 `data/` 运行产物、`.trae/` IDE 文件、迁移备份和一次性排查脚本。
- 完整生产运行仍依赖本机 WSL VoxCPM2、FFmpeg、OpenCLI、模型 API 和 Live2D 环境。
